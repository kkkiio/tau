# PRD: Left Sidebar

## Problem Statement

Pi Web UI 的左侧导航栏承担了过多职责——会话列表、设置入口、主题切换等散落在不同位置。用户经常需要在不同视图间切换，但找不到一致的导航心理模型。设置入口的缺失尤其突出：在聊天头部放一个设置图标显得突兀，不放又让用户不知道该去哪里修改偏好。

## Solution

将左侧导航栏重构为三个功能区域：顶部应用标识，可滚动的内容区（活跃会话摘要 + 历史会话），底部固定工具区（设置）。设置入口固定在左下角、始终可达。聊天头部专注于会话相关的控制（模型选择、思维级别、上下文管理），不再承载应用级操作。

## 交互模型

```
Sidebar                            Main Panel
┌────────────────────┐             ┌────────────────────────┐
│ ☰ Pi Web UI  🔄    │             │ Session Name    model   │
│                    │             │                        │
│ ● Active ─────────│             │    聊天内容（实时）      │
│   streaming...     │             │                        │
│                    │             │                        │
│ ─ History ────────│             │ [prompt input] ✅      │
│   /project-a       │             └────────────────────────┘
│     ● Session 1 ⭐ │ ← 绿点在这
│     Session 2      │
│   /project-b        │
│     Session 3 [选中]│ ← 灰底高亮
│                    │
└────────────────────┘
```

| 操作 | Sidebar 变化 | Main Panel 变化 |
|------|-------------|----------------|
| Pi 在 /project-a 跑 session 1 | 绿点在 Session 1，标题显示 session 1 名 | 实时聊天，input 可用 |
| 点击 history 中的 Session 3 | 绿点不动，Session 3 灰底高亮（选中态） | 展示 Session 3 的历史内容，input 不可用 |
| 点回 Session 1 或"回 live"按钮 | 灰底高亮回到 Session 1，绿点仍在 Session 1 | 恢复实时聊天，input 恢复 |

**核心规则**：
- **绿点** = 真实活跃 session（唯一来源：mirror_sync），不受任何点击影响
- **灰底高亮** = 当前正在展示的 session item（独立于绿点），浏览 history 时跟随所选 item，回到 live 时回到活跃 session
- **Active session 标题** = 永远显示绿点对应的 session 名，浏览历史时不变化
- **Input 区域** = 只在浏览活跃 session 时显示，查看历史时隐藏

## User Stories

### 会话与导航

1. 作为 Pi Web UI 用户，我希望左侧导航栏顶部有一个不随展开/收起而移动位置的 toggle 按钮，这样我始终能在同一个像素位置点击它收起或展开 sidebar。

2. 作为 Pi Web UI 用户，我希望 sidebar 展开/收起时过渡平滑，toggle 不闪烁、内容不闪现。

3. 作为 Pi Web UI 用户，我希望 sidebar 顶部显示当前活跃会话的摘要（连接状态、模型名称），这样不用滚动就能了解会话概况。

4. 作为 Pi Web UI 用户，我希望活跃会话的指示器（绿点）始终指向 Pi 进程当前活跃的 session，不受我在 History 列表中点击其他 session 的影响。

### 历史会话

5. 作为 Pi Web UI 用户，我希望历史会话按项目目录（cwd 最后一段）分组展示，分组标题可点击展开/收起，这样不同项目的历史容易区分，大量 session 时也能管理。

6. 作为 Pi Web UI 用户，我希望每个历史会话 item 上显示会话标题和时间戳，收藏重要的会话以便快速访问。

7. 作为 Pi Web UI 用户，我希望在 History 区域搜索会话，搜索范围覆盖所有历史会话。

8. 作为 Pi Web UI 用户，我希望点击历史会话后可以在聊天区域查看其内容（灰底高亮显示当前正在浏览的 item），但这不改变活跃会话指示器（绿点）。

9. 作为 Pi Web UI 用户，浏览历史会话时我希望聊天区域的 input 区域变为不可用状态（显示提示文字），以免我误以为可以在此继续对话。回到活跃会话后 input 恢复。

### 设置与工具

10. 作为 Pi Web UI 用户，我希望左侧导航栏底部始终能看到设置入口，这样无论我在哪个页面，都能快速找到设置。

11. 作为 Pi Web UI 用户，我希望聊天头部只显示与当前会话相关的控制（模型、思维级别、上下文），这样头部保持简洁，不会和导航栏功能重复。

12. 作为 Pi Web UI 用户，我希望主题切换放在设置内部而不是独立的头部按钮，因为主题是配置偏好而不是日常操作。

### 移动端

13. 作为 Pi Web UI 移动端用户，即使屏幕狭小，我打开侧边栏抽屉时仍然能在底部找到设置，这样设置不会因为空间不足而被隐藏。

### 布局兼容

14. 作为 Pi Web UI 用户，无论当前处于双列还是三列布局模式，我希望设置始终在左侧导航栏底部的固定位置，这样始终能预期在哪里找到它。

## Out of Scope

- 不包含多项目启动功能
- 不支持从 Web UI 切换活跃 session（需在 Pi TUI 中操作）
- 不新增设置类别（超出已有配置需求）
- 不将模型和思维级别控制移入侧边栏
- 不将主题切换提升为头部的独立操作

## Further Notes

- 设置入口的「始终可达」指左侧导航栏可见时底部固定显示；导航栏折叠为抽屉时，设置仍位于抽屉底部
- 后续可在底部工具区增加账户信息、用量统计等持久功能入口
